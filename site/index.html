star Star / 星星 给项目点赞收藏，表示「这个有用」 Star
fork Fork / 复刻 复制别人的仓库到你账号下，可自由修改 Fork

issue Issue / 议题 提 Bug、提需求、讨论问题的工单 Issue
clone Clone / 克隆 把远程仓库完整下载到本机 Clone
commit Commit / 提交 保存一次代码快照，附带说明 Commit
branch Branch / 分支 平行开发线，互不影响 Branch

repository Repository / 仓库 存放项目全部文件与历史的文件夹 Repository

readme README 项目首页说明，第一眼看到的内容 README
release Release / 发行版 打包标记的稳定版本，方便下载 Release
contributor Contributor / 贡献者 参与过项目代码或文档的人 Contributor
watch Watch / 关注 订阅仓库动态通知 Watch

discussions Discussions 仓库内的论坛式讨论区 Discussions
gist Gist GitHub 的代码片段分享 Gist

downstream 下游 从你的仓库 Fork 出去的副本 Downstream
markdown Markdown 轻量标记语言，README 常用 Markdown

🤖 AI & 大模型 (32 条)

llm 大语言模型 读懂并生成人类语言的 AI 模型 LLM
token Token / 词元 模型处理文本的最小单位，影响计费与长度 Token
prompt 提示词 你给 AI 的指令或问题 Prompt
rag RAG / 检索增强 先查资料再回答，减少胡编 RAG
agent Agent / 智能体 能自己规划步骤、调用工具的 AI Agent
mcp MCP AI 连接外部工具和数据的标准「插座」 Model Context Protocol
skills Skills / 技能 教 AI 在特定场景下怎么做事的说明书 Skills
function-calling 函数调用 AI 决定调用哪个 API 并传参数 Function Calling
embedding 向量嵌入 把文字变成数字向量，便于语义搜索 Embedding
fine-tuning 微调 在通用模型上继续训练以适应特定任务 Fine-tuning
hallucination 幻觉 AI 自信地编造不存在的事实 Hallucination

temperature 温度 控制输出随机性，越高越发散 Temperature
system-prompt 系统提示 设定 AI 角色与行为边界的隐藏指令 System Prompt
inference 推理 模型运行生成结果的过程 Inference
training 训练 用数据调整模型参数的学习过程 Training
multimodal 多模态 同时理解文字、图片、音频等 Multimodal
vibe-coding 氛围编程 用自然语言描述意图，让 AI 写大部分代码 Vibe Coding
harness Harness / 驾驭工程 给 AI Agent 设计可控、可验证的工作环境 Harness Engineering
tool-use 工具使用 AI 调用搜索、代码执行等外部能力 Tool Use
chain-of-thought 思维链 让 AI 分步推理，提高复杂题正确率 Chain-of-Thought
guardrails 护栏 限制 AI 输出范围的安全规则 Guardrails
quantization 量化 降低模型精度以减小体积、加速推理 Quantization
distillation 蒸馏 大模型教小模型，保留部分能力 Distillation
a2a A2A 多个 AI Agent 互相通信的协议 Agent2Agent
computer-use Computer Use AI 直接操控鼠标键盘操作电脑 Computer Use
reasoning-model 推理模型 花更多思考步骤换更高正确率 Reasoning Model
context-engineering 上下文工程 设计喂给模型的信息与结构 Context Engineering
evals 评测 系统化测试 AI 输出质量 Evals
grounding 事实锚定 让回答基于可验证事实而非纯生成 Grounding
workflow AI 工作流 多个 AI 步骤串联的自动化流程 AI Workflow
subagent 子 Agent 主 Agent 派生的专项小 Agent Sub-agent
