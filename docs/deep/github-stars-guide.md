# Star 能当简历吗？怎么涨到 1000？

> 5 分钟读完 · 给想涨 Star 的开发者

## Star 是什么？

**Star = 给 GitHub 项目点收藏 + 点赞。**  
表示：「这个对我有用，以后还要找。」

## 500 / 1000 Star 有意义吗？

**有意义，但看「真实度」：**

| 情况 | 面试官怎么看 |
|------|-------------|
| 1000 Star + 活跃 Issue + 真实用户 | 强加分 |
| 1000 Star + 0 Issue + 代码无人 Fork | 可疑 |
| 0 Star + 解决真实问题的好 README | 仍值得做 |

评论区说的「500 Star 去大厂够了」——指的是 **真实影响力**，不是数字游戏。

## 什么项目容易涨 Star？

根据 GitHub 中文区数据（2026）：

| 类型 | 代表 | Star 量级 |
|------|------|-----------|
| 术语 / 科普词典 | computerese-cross-references | 1000+ |
| 5 分钟概念短文 | one-small-step | 6900+ |
| 解决刚需的小工具 | claude-desktop-zh-cn | 987+ |
| Awesome 清单 | 各类 awesome-* | 500–5000 |

**共同点：收藏即价值 — 用户 Star 是为了「以后还要用」。**

## 涨 Star 三板斧（可执行）

### 1. 包装（1 天）

- README 顶部 30 秒看懂
- 中英双语 + 表格速查
- Topics 标签齐全
- Release 发 v1.0.0

### 2. 传播（持续）

- V2EX / 掘金 / 知乎 发教程帖
- 在相关 Issue 下**礼貌**分享（不是 spam）
- 短视频「编程名词扫盲」带链接

### 3. 维护（每周）

- 回复 Issue
- 加新词条 / 修 Bug
- 发小版本 Release

## 绝对不要

- ❌ 买 Star / 互刷群
- ❌ 只 upload 不维护
- ❌ 标题党 README、代码空壳
